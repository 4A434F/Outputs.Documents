# BGOW0044 Document 1041 Mapping Report

Document `1041` is the ENTI/ENT1 interested-entity cancellation letter for contract resolution.

This report has two sections:

1. the fast mapping;
2. the validation tables that show where each decision can be checked in the legacy files and current .NET implementation.

## Fast Mapping

### Selection / Evaluation Fields

- `COD-DOCUMENTO = ENTI` or `ENT1`.
- Legacy selection moves `1041` to `CODBAR-IDENTIF-DOCUM`.
- Legacy layout -> `X014ENTIT1`.

### Displayed Fields

#### Address Window

Mapping:

- `CH-NOME` -> `BGOW0044Contract.BeneficiaryIntervenorName`
- `CH-LOCATION-REF` -> not currently modeled directly
- `CH-MORADA1/2/LOCALIDADE/CPOSTAL fields` -> `not currently modeled as a creditor address object`

Rendered by:

- LetterLayout Address or future creditor/interested-entity address component

#### Subject Block

Mapping:

- `CH-NOME` -> `BGOW0044Contract.BeneficiaryIntervenorName`
- `CH-NUMBER` -> not currently modeled directly
- `CH-MORADA1/2/LOCALIDADE/CPOSTAL fields` -> `not currently modeled as a creditor address object`
- `CL-NOME and client address fields` -> `BGOW0044Contract.Client plus subject/address text`
- `DEB-IBAN / DEB-SWIFT` -> `not currently modeled for this display path`
- `CL-NOME` -> `BGOW0044Contract.Client.Name`
- `CL-NIF` -> `BGOW0044Contract.Client.TaxIdentificationNumber`
- `NR-APOLICE` -> `BGOW0044Contract.Policy.PolicyNumber`
- `DESCR-RAMO-PROD` -> `BGOW0044Contract.Product.BranchProductDescription`
- `risk text from UR-TIPO and related risk fields` -> `BGOW0044Contract.RiskUnit.Description`

Rendered by:

- LetterLayout Subject / SubjectDetails

#### Body Variables

Mapping:

- `DT-ANULACAO` -> `BGOW0044Contract.CancellationDate`

Rendered by:

- document body paragraphs

#### Trace / Signature Context

Mapping:

- `NR-APOLICE` -> `BGOW0044Contract.Policy.PolicyNumber`
- `generated document id` -> `BGOW0044Contract.DocumentIdentification`
- `USERID` -> trace text source in legacy; current templates use static or layout-level trace data
- `COD-ORIGEM` -> trace text source in legacy; current templates use static or layout-level trace data
- `selected ENTI/ENT signer` -> `static SignatureComponent configuration`

Rendered by:

- LetterLayout TraceText / SignatureComponent

## Legacy Layout / PDF Evidence

Uses ENTI formatter `V200-FRMT-ENTI` and page layout `X014ENTIT1`.

## Current Implementation Notes

- Interested-entity address fields are not fully represented by the current contract.
- The document uses the same subject formatter as other BGOW0044 letters, with additional client-address/IBAN behavior in legacy COBOL.
