# BGOW0044 Document 1186 Mapping Report

Document `1186` is the life cancellation/anulment letter FP13 (Beneficiário Interventor com Crédito não CGD).

This report has two sections:

1. the fast mapping;
2. the validation tables that show where each decision can be checked in the legacy files and current .NET implementation.

## Fast Mapping

### Selection / Evaluation Fields

- `COD-DOCUMENTO = FP13`.
- Legacy selection moves `1186` to `CODBAR-IDENTIF-DOCUM`.
- Legacy layout -> `X014FP13T0` and positioning layout `X014FP13P0`.

### Displayed Fields

#### Address Window

Mapping:

- `CH-NOME` -> `BGOW0044Contract.BeneficiaryIntervenorName`
- `CH-* address fields` -> `not currently modeled as a creditor address object`

Rendered by:

- LetterLayout Address or future creditor address component

#### Subject Block

Mapping:

- `VA-NOME-TOMADOR` -> `BGOW0044Contract.PolicyholderName`
- `PS-1-NOME / PS-2-NOME and NIF fields` -> `BGOW0044Contract.InsuredPeople`
- `CH-NOME` -> `BGOW0044Contract.BeneficiaryIntervenorName`
- `VA-PROC-BANC` -> `BGOW0044Contract.BankProcess`
- `NR-APOLICE` -> `BGOW0044Contract.Policy.PolicyNumber`
- `DESCR-RAMO-PROD` -> `BGOW0044Contract.Product.BranchProductDescription`

Rendered by:

- LetterLayout Subject / SubjectDetails

#### Body Variables

Mapping:

- `DT-ANULACAO` -> `BGOW0044Contract.CancellationDate`

Rendered by:

- document body paragraphs

#### Trace / Signature Context

Mapping:

- `NR-APOLICE` -> `BGOW0044Contract.Policy.PolicyNumber`
- `generated document id` -> `BGOW0044Contract.DocumentIdentification`
- `USERID` -> trace text source in legacy; current templates use static or layout-level trace data
- `COD-ORIGEM` -> trace text source in legacy; current templates use static or layout-level trace data
- `selected FP signer` -> `static SignatureComponent configuration`

Rendered by:

- LetterLayout TraceText / SignatureComponent

## Legacy Layout / PDF Evidence

Uses `V2000-FRMT-FPXX` for life cancellation letters. `X014FP13T0` carries the main text; `X014FP13P0` positions the signature/body continuation.

## Current Implementation Notes

- These letters do not use the receipt-list flow used by the lack-of-payment variants.
- FP13 skips the `X014FP00LA` support-line paragraph in legacy COBOL.
