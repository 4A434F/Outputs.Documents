1. the fast mapping;
2. the validation tables that show where each decision can be checked in the legacy files and current .NET implementation.

## Fast Mapping

### Selection / Evaluation Fields

- `COD-DOCUMENTO = FP05`.
- Legacy selection moves `1178` to `CODBAR-IDENTIF-DOCUM`.
- Legacy layout -> `X014FP05T0` and positioning layout `X014FP05P0`.

## Legacy Layout / PDF Evidence

Uses `V1000-FRMT-FPXX` for lack-of-payment life letters. `X014FP05T0` carries the main text; `X014FP05P0` positions the signature/body continuation. `X014REFMB` is written for this FP code.

## Current Implementation Notes

- Life-letter subject data is currently split across generic fields and life-specific fields on `BGOW0044Contract`.
- Receipt rows are rendered for lack-of-payment letters.
- Only FP01, FP02, FP05, and FP07 write the Multibanco layout.
