# BGOW0044 Document 1040 Mapping Report

Document `1040` is the CDC1 cancellation letter used when the cancellation produces a premium amount to pay.

This report has two sections:

1. the fast mapping;
2. the validation tables that show where each decision can be checked in the legacy files and current .NET implementation.

## Fast Mapping

### Selection / Evaluation Fields

- `COD-DOCUMENTO = CDC1`
  - `COD-DOCUMENTO` + legacy evaluation result -> `BGOW0044Contract.DocumentIdentification = "1040"`.
- `DT-SINISTRO` must be empty.
  - `DT-SINISTRO` -> selection-only legacy field. It is not currently modeled on `BGOW0044Contract`.
- `VLR-DIVIDA > 0` selects document `1040`.
  - `VLR-DIVIDA` -> `BGOW0044Contract.DebtAmount`.
- `COD-RAMO(1:4) = 0601` only switches between `X014CDC1TD` and `X014CDC1T4`.
  - `COD-RAMO` -> `BGOW0044Contract.Product.BranchCode`.
  - Meaning: layout position variant only; it does not change the domain meaning of document `1040`.

## PDF Validation

The legacy PDF confirms:

- recipient address is visible in the address window;
- trace/id line contains document identification `1040`;
- title is `ANULAÇÃO DE APÓLICE`;
- letter date is visible;
- subject block contains policyholder, NIF, policy/product, and insured risk;
- body contains cancellation date and premium amount;
- body text says payment will be made through a future Aviso Recibo;
- signature is Pedro Mortágua / DFN.

The PDF does not show the Multibanco payment box.

`ATM-ENTIDADE` and `ATM-REF` exist in the copybook and are copied into temporary fields, but `PGOC114A` only writes `X014REFMB` for `FP01`, `FP02`, `FP05`, and `FP07`. Therefore, those fields remain part of the shared `BGOW0044Contract`, but they are not displayed by FS1040.

## Current Implementation Notes

- FS1040 currently maps to `BGOW0044Contract`.
- The template uses `LetterLayout` for address, subject, date, body, and trace.
- The template does not render `AtmPayment`, matching the 1040 PDF and COBOL flow.
- `DT-SINISTRO` is a legacy selection field. It is not displayed and is not currently present in the .NET contract.
